## Этап 1.1 — Каркас проекта, Docker и базовый API

**Цель:**  
Создать стартовую структуру проекта, docker-инфраструктуру и минимальный API-сервис для локального запуска через Docker Compose.

**Что просили у ИИ:**  
Сгенерировать структуру проекта с папками `src/api`, `src/bot`, `docs`, `tests`, подготовить `docker-compose.yml`, `Dockerfile.api`, `Dockerfile.bot`, `.env.example`, `.gitignore`, `README.md`, `docs/dev_log.md`, а также реализовать базовое FastAPI-приложение с маршрутами `/health` и `/ping`.

**Результат:**  
Создан каркас проекта, настроен Docker Compose с сервисами `dm-api` и `dm-bot`, добавлен базовый API, подготовлены начальные файлы инфраструктуры и документации.

**Prompt:**
Мы начинаем реализацию проекта DocuMind. Работаем строго по MVP и модульной архитектуре.

Задача этого шага: создать каркас проекта, docker-инфраструктуру и базовый API-сервис.
Ничего лишнего не реализовывать.

Важно:
- использовать нейминг Docker с префиксом dm
- контейнеры: dm-api, dm-bot
- сеть: dm-net
- проект должен запускаться локально через docker compose
- после выполнения шага нужно обновить:
  1) docs/dev_log.md
  2) .gitignore
  3) README.md

Нужно создать/обновить структуру проекта примерно так:

pdf_contract_analyzer/
  docs/
    idea.md
    tz.md
    prompts.md
    dev_log.md
  src/
    api/
      main.py
      __init__.py
    bot/
      __init__.py
  tests/
  .env.example
  .gitignore
  README.md
  docker-compose.yml
  Dockerfile.api
  Dockerfile.bot
  requirements.api.txt
  requirements.bot.txt

Требования:

1. docker-compose.yml
- сервис dm-api
- сервис dm-bot (пока можно как заглушку-команду sleep или минимальный python entrypoint, если бот еще не реализован)
- сеть dm-net
- env_file: .env
- нормальные restart policy
- без слишком общих container_name

2. API
- FastAPI приложение в src/api/main.py
- endpoints:
  - GET /health -> {"status": "ok"}
  - GET /ping -> {"message": "pong"}
- запуск через uvicorn
- минимально чистая структура

3. Dockerfile.api
- Python 3.12-slim
- установка зависимостей из requirements.api.txt
- запуск API

4. Dockerfile.bot
- Python 3.12-slim
- установка зависимостей из requirements.bot.txt
- пока только базовый entrypoint-заглушка, чтобы контейнер поднимался без ошибок

5. .env.example
Добавить переменные-заглушки:
- API_HOST=0.0.0.0
- API_PORT=8000
- TELEGRAM_BOT_TOKEN=
- PROXYAPI_API_KEY=
- GOOGLE_OAUTH_CLIENT_SECRET_JSON=
- GOOGLE_DRIVE_FOLDER_ID=

6. .gitignore
Добавить все базовые служебные исключения для Python, Docker, env, OCR/временных файлов, токенов, IDE и локальных артефактов.

7. README.md
Пока только базовая версия:
- название проекта
- краткое описание
- текущий статус: Stage 1 / project skeleton
- инструкция запуска через docker compose

8. docs/dev_log.md
Создать и добавить первую запись:
- создан каркас проекта
- добавлены Docker-сервисы
- поднят базовый API
- подготовлены .env.example, README.md, .gitignore

9. Качество
- не добавлять пока бизнес-логику

## Этап 4.1.4 — Переход на span-based masking

**Цель:**

при замене возникают артефакты:

# Формулировка идеи проекта

## Название проекта (рабочее)

**DocuMind — AI-сервис для обработки PDF-договоров**

---

## Описание проекта

DocuMind — это сервис для автоматизированной обработки договоров, который позволяет:

- извлекать текст из PDF и DOCX документов  
- выполнять OCR для сканированных документов  
- деперсонализировать чувствительные данные  
- анализировать документы с помощью LLM  
- сравнивать версии договоров  
- формировать структурированные результаты и отчеты  
- генерировать итоговые `.docx` файлы  

Сервис реализует единый pipeline обработки документов и предоставляет доступ через два интерфейса:

- Web UI  
- Telegram-бот  

Дополнительно реализована интеграция с Google Drive для сохранения результатов обработки.

---

## Основная цель проекта

Создать MVP-сервис, который автоматизирует обработку договоров и демонстрирует полный цикл обработки документов:

**от загрузки → до получения структурированного результата**

---

## Ценность проекта (практическое применение)

Сервис может использоваться для:

- ускорения анализа договоров  
- извлечения ключевых условий  
- предварительной оценки содержания документа  
- автоматизации подготовки документов  
- снижения ручной работы  

---

## Ключевая особенность проекта

### 1. Безопасность данных
- деперсонализация выполняется **до отправки в LLM**
- исходные данные не передаются во внешние сервисы

### 2. Единый pipeline
- Web и Telegram используют одну backend-логику
- одинаковые результаты при разных интерфейсах

### 3. Поддержка OCR
- корректная обработка сканированных документов

---

## Ограничения по безопасности данных

- персональные данные не сохраняются  
- обработка выполняется в runtime  
- используется masking перед LLM  

---

## Функциональные возможности (актуализировано)

- загрузка PDF и DOCX  
- извлечение текста (включая OCR)  
- деперсонализация данных  
- анализ через LLM  
- сравнительный анализ  
- генерация `.docx`  
- логирование этапов обработки  
- отображение статусов пользователю  
- OAuth авторизация Google Drive  
- сохранение результатов в Google Drive  

---

## Ограничения MVP (актуализировано)

- обработка **1 запроса за раз (lock-механизм)**  
- отсутствие очереди задач  
- нет хранения истории  
- зависимость от LLM API  
- ограниченная производительность (VPS ~1GB RAM)  

---

## Потенциал развития

Проект спроектирован как расширяемый и может быть развит до:

- SaaS-сервиса  
- системы анализа различных типов документов  
- инструмента для юридической автоматизации

#Формулировка идеи проекта
##Название проекта (рабочее)

**AI-сервис для обработки PDF-договоров**
---

##Описание проекта

Проект представляет собой сервис, который позволяет обрабатывать договоры в формате PDF: извлекать из них текст, анализировать содержимое с помощью AI и формировать структурированные результаты.

Сервис поддерживает несколько режимов работы:

- распознавание и структурирование договора  
- аналитический обзор содержания договора  
- сравнительный анализ двух договоров  

Сервис выполняет локальную предобработку документа, включая деперсонализацию чувствительных данных, после чего передает обезличенный текст в языковую модель для анализа. На основе полученных данных формируются итоговые артефакты: структурированные данные, краткий аналитический отчет и файл документа в формате .docx.

Сервис поддерживает обработку как PDF-документов с текстовым слоем, так и сканированных документов за счет использования OCR.

Дополнительно предусмотрена возможность сохранения результатов обработки в Google Drive.

---

##Основная цель проекта

Создать минимально жизнеспособный сервис (MVP), который автоматизирует обработку PDF-договоров, сокращает объем ручной работы с документами и ускоряет получение структурированной информации и аналитического обзора.

---

##Ценность проекта (практическое применение)

Сервис может использоваться для:

- быстрого извлечения ключевых данных из договоров
- ускорения подготовки новых документов на основе существующих
- первичного анализа содержания договора
- снижения количества ручной работы при обработке документов

---

#Ключевая особенность проекта
Сервис использует гибридный подход к обработке данных:

- перед отправкой в LLM выполняется локальная деперсонализация документа
- персональные и чувствительные данные заменяются на маркеры или ролевые обозначения
- исходные данные не передаются во внешние сервисы

---

##Ограничение по безопасности данных

Сервис не хранит персональные данные пользователей.

Обработка выполняется в stateless-режиме:
данные загружаются, обрабатываются в памяти и не сохраняются после завершения обработки.

---

##Функциональные возможности (уровень идеи)

- загрузка PDF или DOCX-документов  
- извлечение текста из PDF (включая OCR для сканированных документов)  
- локальная деперсонализация данных  
- анализ текста с помощью LLM  
- извлечение ключевых параметров договора  
- генерация краткого аналитического отчета  
- сравнительный анализ двух договоров  
- формирование .docx-файла  
- опциональное сохранение результатов в Google Drive  
- возможность выбора сценария обработки через интерфейс Telegram-бота  

---

##Ограничения MVP
- поддержка одного типа документов (договоры)
- отсутствие хранения истории обработки
- отсутствие полной юридической валидации
- аналитический отчет носит справочный характер и не является юридическим заключением
